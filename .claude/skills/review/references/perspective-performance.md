# Performance Perspective Checklist

パフォーマンス観点でのレビューチェックリスト。
DB アクセス、メモリ使用量、レスポンス時間に焦点を当てる。

## Must Fix

### N+1 クエリ
- [ ] ループ内で DB クエリを実行していないか
- [ ] 関連エンティティの取得が一括取得（JOIN / IN）になっているか
- [ ] バッチ処理で1件ずつ DB アクセスしていないか

### リソースリーク
- [ ] DB コネクション、ストリーム、ファイルハンドルが確実にクローズされているか
- [ ] try-with-resources が適切に使用されているか
- [ ] 大量データを一括でメモリにロードしていないか

### 無限ループ・再帰
- [ ] ループの終了条件が必ず満たされるか
- [ ] 再帰の深さに上限があるか

## Should Fix

### ページネーション
- [ ] リスト取得 API にページネーションが実装されているか
- [ ] 取得件数の上限が設定されているか
- [ ] `SELECT *` ではなく必要なカラムのみ取得しているか

### クエリ最適化
- [ ] 不要な JOIN が含まれていないか
- [ ] WHERE 句のカラムにインデックスが存在するか（設計書と照合）
- [ ] ORDER BY のカラムにインデックスが対応しているか

### キャッシュ活用
- [ ] 頻繁に参照されるマスタデータのキャッシュを検討しているか
- [ ] キャッシュの無効化タイミングが適切か

### バッチ処理
- [ ] 大量データ処理時にチャンク分割が行われているか
- [ ] トランザクション範囲が適切か（長すぎるトランザクションの回避）

## Suggestion

### 文字列操作
- [ ] ループ内での文字列結合に `StringBuilder` を使用しているか
- [ ] 不要な `toString()` 呼び出しがないか

### コレクション操作
- [ ] 適切なコレクション型を使用しているか（検索頻度に応じた `List` / `Set` / `Map` の選択）
- [ ] Stream API の中間操作が過度に長くないか

### 非同期処理
- [ ] 独立した処理を並列実行できる余地がないか
- [ ] 重い I/O 処理の非同期化を検討しているか

### ログ出力
- [ ] DEBUG レベルのログ出力にガード条件があるか（`if (log.isDebugEnabled())`）
- [ ] ループ内で大量のログ出力を行っていないか
