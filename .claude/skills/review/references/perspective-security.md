# Security Perspective Checklist

セキュリティ観点でのレビューチェックリスト。
OWASP Top 10 に基づく汎用チェック項目。

## Must Fix

### インジェクション
- [ ] SQL インジェクション: ORM のバインド変数 / パラメータ化クエリを使用しているか（文字列結合での SQL 組み立てがないか）
- [ ] OS コマンドインジェクション: 外部入力をシェルコマンドに渡していないか
- [ ] LDAP / XPath インジェクション: 該当する場合、入力のエスケープが行われているか
- [ ] ログインジェクション: ユーザ入力をログ出力する場合、改行文字のサニタイズが行われているか

### 認証・認可
- [ ] 認証バイパスの可能性がないか（エンドポイントの認証設定漏れ）
- [ ] 権限チェックの欠如がないか（認証済みユーザの権限検証）
- [ ] トークン（JWT 等）の検証が適切か（署名検証、有効期限チェック）

### 秘密情報
- [ ] ハードコードされたパスワード・API キー・秘密鍵がないか
- [ ] 設定ファイルに平文のクレデンシャルが含まれていないか
- [ ] ログに秘密情報が出力されていないか

### データ検証
- [ ] 外部入力に対するバリデーションが行われているか（型、長さ、範囲、形式）
- [ ] ファイルアップロード時の検証（サイズ、拡張子、MIME タイプ）

## Should Fix

### 入力検証
- [ ] リクエストパラメータのサニタイズが適切か
- [ ] パスパラメータに対するパストラバーサル対策があるか
- [ ] 数値パラメータの範囲チェックが行われているか

### セッション管理
- [ ] セッション固定攻撃への対策があるか
- [ ] セッションタイムアウトが設定されているか

### 暗号化
- [ ] 非推奨の暗号アルゴリズムを使用していないか（MD5, SHA-1 等）
- [ ] 乱数生成にセキュアな乱数生成器を使用しているか

### エラーハンドリング
- [ ] スタックトレースがレスポンスに含まれていないか
- [ ] エラーメッセージに内部情報（テーブル名、カラム名、パス等）が漏洩していないか

## Suggestion

### 情報漏洩
- [ ] エラーレスポンスが過度に詳細でないか（攻撃者への情報提供を最小限に）
- [ ] HTTP レスポンスヘッダに不要なサーバ情報が含まれていないか

### 依存ライブラリ
- [ ] 既知の脆弱性を持つライブラリバージョンを使用していないか
- [ ] 不要な依存が追加されていないか

### セキュリティヘッダ
- [ ] Content-Type の明示的指定が行われているか
- [ ] CORS 設定が適切か（ワイルドカード `*` を避ける）
